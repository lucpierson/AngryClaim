AngryClaim : add a REST binding

Goal : this document details the steps required to add and test a REST binding to the existing AngryClaim application

Required classes and artefacts

  • org.jboss.demo.fsw.angrytweet.rest.ServiceTicketResource : annotated interface which describes the resources exposed by the REST binding.
    The interface exposes a method to submit new tickets through POST.

  • org.jboss.demo.fsw.angrytweet.rest.ServiceTicket : POJO to which the JSON payload will be marshalled to.

  • org.jboss.demo.fsw.angrytweet.rest.ServiceTicketTransformer : transformer class to transform from org.jboss.demo.fsw.angrytweet.rest.ServiceTicket to the canonical org.jboss.demo.fsw.angrytweet.model.ProviderServiceTicket object.

  • ChannelOut.drl : a new rule was added to handle requests submitted through the REST binding

Add the REST binding

  • In JBDS, open the switchyard.xml configuration file with the Switchyard Visual Editor

  • In the canvas, right-click the service icon (the little green icon on the left) of the TicketEnricher component and select Promote Service.

  • In the Promote Component Service Window, select ESB as interface type.

  • Click on Browse to define the input type. Enter org.jboss.demo.fsw.angrytweet.rest.ServiceTicket in the Input Type field. Leave the other fields blank. Click OK.

  • Enter ServiceTicket in the Service Name field. Click Finish.

  • Hover over the newly created binding shape and click on the Binding icon. In the list select REST.

  • In the REST Binding Details Window specify rest as Context Path. Click on the Add button and add org.jboss.demo.fsw.angrytweet.rest.ServiceTicketResource to the list of interfaces. Click Finish.

Test the REST binding

  • The new REST binding can be tested with SOAPUI or curl. The REST service URL is localhost:8080/rest/serviceticket

  • Build and deploy the project.

  • To test with curl, open a terminal and issue the following command 

    $ cd ./etc/rest/
    $ curl -X POST -H "Content-Type: application/json" -d @ticket.json "localhost:8080/rest/serviceticket"

    ticket.json is a file with the JSON payload. An example (see ./etc/rest/):

    {
        "customer" : "JohnDoe",
        "service" : "cable",
        "problem" : "service down",
        "urgent" : "1"
}
